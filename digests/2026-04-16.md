## Deep Dive of the Day

**Topic: OpenAI Agents SDK — Separating the Control Harness from the Compute Layer**

This is the highest-signal technical pattern to understand for a FAANG AI agent interview right now. 
OpenAI's new architecture separates the control harness from the compute layer, keeping credentials entirely out of the environments where model-generated code executes; by isolating the execution layer, an injected malicious command cannot access the central control plane, protecting the wider corporate network from lateral movement attacks.
 This directly addresses two canonical interview topics at once: **durable execution** and **agent security**. 
Long-running tasks often fail midway due to network timeouts, container crashes, or API limits — if a complex agent takes twenty steps to compile a report and fails at step nineteen, re-running the entire sequence burns expensive compute; under the new architecture, losing the sandbox container does not mean losing the entire operational run.
 For deeper preparation, explore: (1) checkpointing strategies in LangGraph's `SqliteSaver` and `PostgresSaver`; (2) the Manifest abstraction pattern for portable workspace descriptions; (3) MCP as a trust boundary between the agent harness and external tool surfaces; and (4) how the OWASP Agentic Top 10 maps to each layer of a sandboxed agent architecture.
