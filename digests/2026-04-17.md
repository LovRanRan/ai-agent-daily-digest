## 3. Engineering Practice / Technical Blogs

- **[Anthropic Engineering — Three-Agent Harness: Planner / Generator / Evaluator](https://www.anthropic.com/engineering)**. 
Anthropic published a three-agent harness architecture designed to keep autonomous coding agents productive across multi-hour sessions; the system splits work among a planner, a generator, and an evaluator, using context resets instead of compaction to prevent drift and premature task termination.
 
As the context window fills, coherence degrades, and even with compaction models exhibit what Anthropic calls "context anxiety," where they start wrapping up work prematurely as they approach perceived context limits.
 
The planner decomposes a product spec into discrete, tractable chunks self-contained enough for a fresh agent to execute cold; the generator builds the code within a single context window per task, then hands off structured artifacts (JSON feature specs, commit-by-commit progress, claude-progress.txt) to the next session; the evaluator grades output using predefined criteria and for frontend work navigates live pages using Playwright MCP.
 Takeaway: hard context resets + structured handoff artifacts beat compaction for multi-hour agent runs.

- **[LangChain — Continual learning for AI agents](https://www.langchain.com/blog/continual-learning-for-ai-agents)** (Harrison Chase). 
Most discussions of continual learning in AI focus on updating model weights, but for AI agents learning can happen at three distinct layers: the model, the harness, and the context — and understanding the difference changes how you think about building systems that improve over time.
 
As harnesses have become more popular, papers like Meta-Harness: End-to-End Optimization of Model Harnesses show the core idea — run the agent over a bunch of tasks, evaluate them, store logs into a filesystem, then run a coding agent to look at traces and suggest changes to the harness code.
 Takeaway: a clean mental model for deciding where to invest your eval/optimization budget.

- **[LangChain + MongoDB Partnership — agent stack on operational data](https://www.langchain.com/blog/announcing-the-langchain-mongodb-partnership-the-ai-agent-stack-that-runs-on-the-database-you-already-trust)** (April 16). 
Production agents need durable state; in the default architecture each deployment requires a dedicated Postgres instance to handle the high-volume checkpoint writes that power multi-turn memory, human-in-the-loop workflows, time-travel debugging, and fault-tolerant execution, so infrastructure scales linearly with the number of agents shipped, whereas the MongoDB Checkpointer collapses that to a fixed cost handling checkpoint and memory writes across all deployments in a single shared cluster.
 War-story takeaway: checkpointer infra cost is the hidden tax at N agents in production.

- **[Epsilla — The GAN-Style Agent Loop](https://www.epsilla.com/blogs/anthropic-harness-engineering-multi-agent-gan-architecture)**. 
The new frontier is "Harness Engineering"—building structured environments for AI agents to operate reliably; Anthropic's architecture is inspired by Generative Adversarial Networks, separating a Generator from a skeptical Evaluator to create a feedback loop that overcomes AI's inability to self-critique, and for subjective tasks the Evaluator uses tools like Playwright MCP to interact with live outputs.
 
The quality gap was vast — tasked with a 2D retro game-making tool, a single agent produced a barely-functional prototype in 20 minutes for $9, while the full harness ran 6 hours, cost $200, and delivered a polished, genuinely useful application — not a 2x improvement but a phase change.

## Deep Dive of the Day

**Topic: Harness engineering — context resets vs. compaction for long-horizon agents.**

The single most interview-relevant concept converging across today's news is the **harness layer** sitting between the model and the world, and the specific failure mode of **context anxiety**. 
As the context window fills, coherence degrades; even with compaction, models exhibit "context anxiety" where they wrap up work prematurely as they approach perceived context limits, and Rajasekaran found Claude Sonnet 4.5 exhibited this strongly enough that compaction alone wasn't sufficient.
 FAANG agent interviews increasingly probe: "your agent has a 200k context, the task needs 6 hours — what do you do?" The modern answer is no longer "summarize and hope" but **structured handoff via planner/generator/evaluator with hard resets**, each session re-reading state from a filesystem artifact (claude-progress.txt, JSON specs) rather than inheriting chat history. Directions worth digging deeper: (1) the Planner/Generator/Evaluator split and how it echoes GAN training loops; (2) how MCP tools like Playwright let the Evaluator ground itself in live execution rather than self-critique; (3) the trade space between MemPalace's "store everything verbatim" and claude-mem's "AI-compress" memory strategies; and (4) OpenAI's new Manifest spec as the portable workspace primitive that sits beneath all of this. Know these four, and you can answer almost any long-horizon agent design question.
