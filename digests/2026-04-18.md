## Deep Dive of the Day

**Topic: The three-layer learning model for agents (Model / Harness / Context)**, from Harrison Chase's post.

This is the single most interview-relevant idea on today's digest. 
The model is the weights themselves; the harness is the code that drives the agent plus any instructions or tools always part of the harness; context is additional configuration like instructions and skills that lives outside the harness
. FAANG interviewers increasingly push past "how does RAG work" into "how does your agent get better over time without retraining" — and this framework gives you a crisp decomposition to answer that. Directions worth digging into: (1) **Harness optimization** via meta-learning over trace logs (
the Meta-Harness paper's core idea is that the agent runs in a loop over tasks, evaluates them, stores logs to a filesystem, then runs a coding agent over those traces to suggest changes to the harness code
); (2) **Context/skills** as portable learning units — the same abstraction Anthropic and OpenAI have both converged on; (3) **Tenant-level vs. agent-level memory** trade-offs, including the cost/privacy implications of per-user LoRAs vs. per-user Markdown memory files. Be ready to map any real agent system (Claude Code, Codex, Cursor, Devin) onto these three layers on a whiteboard.
