## Deep Dive of the Day

**Topic: The three-layer learning model for agents (Model / Harness / Context), and why "progressive tool discovery" is the concrete production technique to master.**

Harrison Chase's framing — that agents can improve at the model, harness, or context layer — is the single most useful mental scaffold for an agent engineer interview, because it maps directly onto the design tradeoffs FAANG teams actually debate: do we fine-tune, re-engineer the loop, or just add memory/skills? Pair this with the MCP production pattern of 
deferring tool loading until the model actually needs a tool — loading a minimal catalog first, then the specific tool on request
, and you have a concrete architectural answer to the most common failure mode in multi-tool agents: context-window bloat and tool-selection confusion. Interviewers increasingly probe: "Your agent has 100 tools, how do you keep it fast and accurate?" — the answer combines (a) harness-level progressive discovery, (b) context-level skills for domain knowledge, and (c) observability via traces that feed evals. Directions worth digging deeper into: the Meta-Harness paper on end-to-end harness optimization referenced by LangChain; Anthropic's open Agent Skills spec and how it composes with MCP tools; deterministic-harness patterns like Archon's YAML workflows + isolated git worktrees; and the emerging "stateless Streamable HTTP + session migration" design in the MCP 2026 roadmap for horizontally scaling tool servers. Mastering this layering plus one real production war story (e.g., 
93% reduction in time-to-root-cause on debugging workflows using coordinated multi-agent execution
) gives you both the system-design vocabulary and the concrete numbers interviewers look for.
