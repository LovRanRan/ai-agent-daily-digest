## Deep Dive of the Day

**The three-layer model of agent learning: Model / Harness / Context** (from Harrison Chase's post).

Why it matters for FAANG interviews: this framework gives you a principled answer to the classic system-design question "how would you make this agent get better over time?" Most candidates conflate fine-tuning with "learning," but 
most discussions of continual learning focus on updating model weights, while for agents, learning can happen at three distinct layers: the model, the harness, and the context; understanding the difference changes how you think about building systems that improve over time
. Directions to dig deeper: (1) **Harness optimization** — 
Meta-Harness: End-to-End Optimization of Model Harnesses — the agent runs in a loop over tasks, evaluates them, stores logs into a filesystem, then runs a coding agent to look at these traces and suggest changes to the harness code
; this is the research frontier. (2) **Context learning granularity** — 
learning context can be done at the agent level (persistent memory that updates its own configuration over time, e.g., OpenClaw's SOUL.md), but is more commonly done at the tenant level (user, org, team)
. (3) Tie this to production signals like **TBT (Time Between Turns)** for prompt caching and to memory architectures like Supermemory — interviewers love when you connect abstract frameworks to measurable operational metrics.
