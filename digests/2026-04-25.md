## Deep Dive of the Day

**Pick: Harness virtualization (Brain / Hands / Session) from Anthropic's Managed Agents.**

This is the single highest-leverage concept on today's feed for a FAANG agent-engineer interview. The reason is that it cleanly answers the question every interviewer eventually asks: "How do you build agent infrastructure that doesn't rot when models improve?" 
Anthropic noticed that while building the harness for Claude Sonnet 4.5, the model suffered from "context anxiety," so they added context-reset logic. But when the same harness ran on the more capable Claude Opus 4.5, those resets became dead weight — the stronger model no longer needed them, yet the harness was actively holding it back. This is the fundamental dilemma of the harness: it encodes assumptions about the current model's capabilities, and those assumptions rot quickly as models evolve.
 The OS-style answer — 
a meta-harness approach where multiple agent workflows run on a shared execution substrate that handles secure sandboxing, credential management, session continuity, and observability
 — is the same separation-of-concerns pattern FAANG interviewers prize.

Directions to dig deeper: (1) **Context engineering as an API contract** — study `getEvents()` and how durable session logs decouple memory from the model's context window; (2) **Credential isolation** — 
MCP tools called via a dedicated proxy that fetches credentials from a vault; the harness is never made aware of any credentials
; (3) **Generator/Evaluator (GAN-style) loops** — 
Anthropic's long-running harness work is explicit that separating generation from evaluation into distinct agents outperforms self-evaluation, because agents reliably skew positive when grading their own work
; (4) compare and contrast with LangGraph's durable execution model and OpenAI's WebSocket Responses API for state caching. Be ready to whiteboard the three-layer split and discuss what state lives where, who owns retries, and how the harness survives a model swap.
