## Deep Dive of the Day

**Topic: Harness design and the Session/Harness/Sandbox abstraction for long-running agents.**

This is the single highest-leverage interview topic right now. 
Long-horizon tasks often exceed the length of Claude's context window, and the standard ways to address this all involve irreversible decisions about what to keep
, which is why 
Anthropic's Managed Agents introduces a session as a context object that lives outside the model's context window, durably stored in a session log, with a getEvents() interface that allows the brain to interrogate context by selecting positional slices of the event stream
. This matters for FAANG-style system design rounds because it cleanly separates three concerns interviewers love to probe: durable state (session/event log), reasoning policy (the harness, which can do compaction, progressive disclosure, or context firewalls), and tool execution boundary (sandbox + credential vault). 
The complementary GAN-style pattern — splitting a Generator agent that creates output from a skeptical Evaluator agent that critiques against strict criteria, creating an adversarial feedback loop
 — is the second half of the same story: how do you make a long-running loop self-correcting without a human in every step? Worth digging deeper into: (1) prompt-cache-aware context engineering and how it interacts with idle-session eviction (the Claude Code post-mortem is a goldmine); (2) credential-isolation patterns for MCP via proxy + vault rather than passing tokens to the agent; (3) handoff-artifact design (e.g., `claude-progress.txt`, structured TODO files) versus pure compaction; and (4) when context resets beat compaction, and why "context anxiety" emerges as models approach their window limit.
