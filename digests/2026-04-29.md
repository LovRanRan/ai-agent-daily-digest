## Deep Dive of the Day

**Topic: Session/event-log as a first-class context primitive (Anthropic Managed Agents + AWS Bedrock Managed Agents convergence).**

The single most interview-relevant idea today is the architectural separation now visible in both Anthropic's and AWS's Managed Agents launches: a durable, queryable **session event log** that lives *outside* the model's context window, with a thin **harness** that decides which slices to inject back. 
The two concerns — recoverable context storage in the session and arbitrary context management in the harness — are deliberately separated because the system can't predict what specific context engineering an agent will need.
 This matters for FAANG interviews because long-horizon agents (multi-hour coding, computer use, research) hit the same wall — context-window exhaustion, "context anxiety," irreversible compaction — and every major lab is now converging on the same answer: treat the session as a replayable event stream, not a chat history. Worth digging into: (1) Anthropic's three-agent (planner/generator/evaluator) harness with hard context resets between phases; (2) the trade-offs between summarization, compaction, and `getEvents()`-style positional slicing; (3) how identity, credential vaulting (MCP tokens via proxy), and audit logs hang off the same session abstraction; and (4) prompt-cache hit-rate optimization as a first-class harness concern. If you can whiteboard the session/harness/sandbox split and explain why "context resets beat compaction" for multi-hour tasks, you'll be speaking the same language as the teams actually shipping these systems.
