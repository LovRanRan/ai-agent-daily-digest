## Deep Dive of the Day

**Topic: Harness Engineering — context resets, generator/evaluator splits, and the "Bitter Lesson" applied to agent scaffolding.**

This is the single most asked-about topic in current FAANG agent interviews because it directly touches system design, evaluation, and production reliability. The key insight: 
Agent = Model + Harness; if you're not the model, you're the harness — every piece of code, configuration, and execution logic that isn't the model itself
. Concretely, you should be able to whiteboard (a) why 
compaction summarizes earlier conversation in place but doesn't give the agent a clean slate so context anxiety can persist, while a context reset provides a clean slate at the cost of needing a handoff artifact with enough state for the next agent
, and (b) why a separate evaluator agent — ideally armed with a real tool like Playwright MCP — beats self-evaluation for subjective quality. Directions worth digging deeper into: (1) **handoff artifact design** (`claude-progress.txt`, sprint contracts, structured task lists), (2) **GAN-style multi-agent topologies** including the Magentic-One and supervisor patterns now stable in 
Microsoft Agent Framework 1.0's sequential, group chat, and Magentic-One orchestration patterns
, and (3) **harness-layer interop** — MCP for tools vs. A2A for agents — since 
A2A v1.0 was formally released following the MCP Dev Summit North America (April 2–3 in New York), marking the protocol's transition from draft to production standard
. Expect interviewers to probe failure modes (context anxiety, premature completion, evaluator collusion) and require you to propose concrete mitigations rather than abstract principles.
