## 3. Engineering Practice / Technical Blogs

- **[The Agent Harness: What It Is, Why It Matters, and How to Build One from Scratch](https://atalupadhyay.wordpress.com/2026/05/02/the-agent-harness-what-it-is-why-it-matters-and-how-to-build-one-from-scratch/)** (Atal Upadhyay, today). Core takeaway: a from-scratch tour of the nine pillars of a production harness, with concrete pitfalls — e.g., 
changing dynamic prompt order breaks prefix caching and causes a 3-5x latency spike, so always keep static core first
, and 
hooks (pre-tool/post-tool with structured JSON exit codes 0=allow/1=deny/2=modify) are what enable compliance, RBAC integration, rate limiting, and observability
.

- **[Anthropic Engineering — Harness design for long-running application development](https://www.anthropic.com/engineering/harness-design-long-running-apps)** (referenced in today's Anthropic engineering thread on Managed Agents). Core takeaway: 
harnesses encode assumptions about what Claude can't do on its own, and those assumptions need to be frequently questioned because they go stale as models improve — for example, Claude Sonnet 4.5 used to wrap up tasks prematurely as it sensed its context limit ("context anxiety"), addressed by adding context resets to the harness
.

- **[LangChain — Deep Agents v0.5](https://blog.langchain.com/deep-agents-v0-5/)**. Core takeaway: 
Deep Agents can now delegate work to remote agents that run in the background — async subagents return a task ID immediately and execute independently on a remote server, instead of blocking the supervisor like inline subagents
, with 
Agent Protocol as the wire format so you can point a Deep Agent at any Agent Protocol-compliant server, including any agent deployed with LangSmith or a custom FastAPI service
.

- **[Anthropic's Harness Direction: Agent Infrastructure Is Becoming an Agent OS](https://www.knightli.com/en/2026/04/10/anthropic-harness-agent-os/)**. Core takeaway: 
agent infrastructure is becoming a lightweight Agent OS whose job is not to hard-code today's best workflow but to define long-lived system abstractions
, with the critical design rule that 
session is not model context — session should be a queryable, replayable, and recoverable event log, not a direct history dump into the model
.

- **[HumanLayer — Skill Issue: Harness Engineering for Coding Agents](https://www.humanlayer.dev/blog/skill-issue-harness-engineering-for-coding-agents)**. Core takeaway: 
stuffing every instruction and tool into the system prompt makes the agent worse and burns the instruction budget before work even starts; Skills solve this through progressive disclosure — the agent only gets specific instructions, knowledge, or tools when it decides it needs them
, and 
if an MCP server duplicates functionality available as a CLI well-represented in training data, just prompt the agent to use the CLI for things like GitHub, Docker, or most databases
.

## Deep Dive of the Day

**Topic: Harness Engineering and the "Session ≠ Model Context" abstraction.**

For a FAANG agent-engineer interview, the single most leveraged concept right now is the harness — the orchestration layer between the model and the world. Today's signal is unusually clear: Anthropic's Managed Agents ships 
stable interfaces for sessions, harnesses, and sandboxes
, OpenAI's Codex adds 
persisted /goal workflows with create/pause/resume/clear
, and LangChain's Deep Agents v0.5 standardizes 
async remote subagents over Agent Protocol with task IDs and independent execution
. All three converge on the same architectural insight from the knightli piece: 
session must be a queryable, replayable, recoverable event log — not a raw history dumped into the model
. Worth digging deeper into: (1) the planner/generator/evaluator (GAN-style) split and why 
separating generation from evaluation outperforms self-evaluation because agents reliably skew positive when grading their own work
; (2) context-reset vs. compaction strategies for multi-context-window tasks; (3) progressive disclosure of skills/tools and prefix-caching pitfalls in dynamic system prompts; (4) hook protocols (pre-tool/post-tool, allow/deny/modify) as the enterprise security boundary. Be ready to whiteboard a harness with these layers labeled — that is exactly the system-design question this trend is pushing into FAANG loops.
